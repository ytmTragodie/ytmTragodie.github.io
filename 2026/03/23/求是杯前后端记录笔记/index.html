代码上传至Github

在 GitHub 生成 Personal Access Token

  1. 登录 GitHub
  2. 点击右上角头像 → Settings
  3. 左侧菜单最底部 → Developer settings
  4. 左侧 → Personal access tokens → Tokens (classic)
  5. 点击 Generate new token (classic)
  6. 填写 Note（如 server-deploy），勾选 repo 权限
  7. 滚动到底部点击 Generate token
  8. 复制生成的 token（只会显示一次，如 ghp_xxxxxxxxxxxx）

创建并设置服务器

选 2核(vCPU) 4GiB 内存 的经济型 e 实例

为什么？

  • 内存足够：Spring Boot + Docker 运行时，系统本身占用约 300-500MB，Docker 守护进程约 100MB，你的应用限制在 256-512MB 堆内存，加起来 1.5GB 左右。4GB 内存有充足余量，不会因内存不足导致容器被杀死。

推荐选择：Ubuntu 22.04 64位

  • Ubuntu：Docker 官方文档推荐，社区支持最好，遇到问题容易搜到解决方案。

在服务器上克隆仓库并构建docker镜像

克隆项目

如果没有Git，则先下载Git

在服务器上创建好目录，克隆代码

git clone https://{刚刚获得的Github Personal Token}@github.com/{用户名}/{项目名}.git

构建Docker 镜像

  • cd 到dockerfile所在位置

  • 构建docker 镜像

    这个过程会

    • 下载 Maven 和 JDK 镜像（第一次需要几分钟）
    • 编译打包你的 Spring Boot 项目
    • 生成名为 my-backend 的镜像

运行容器

  • -d：后台运行
  • --name my-app：容器命名
  • -p 8080:8080：主机端口映射到容器端口
  • --restart unless-stopped：服务器重启后自动启动容器
